20260714v1.0A 新增文件初版 — 后台「充值提款」新增「eWallet绑定手机门号」开关—
需求目的
前台用户在绑定eWallet(含GCash、Maya) 时，「手机号码」栏位目前须用户自行手动输入。本次需求新增后台开关「eWallet绑定手机门号」，开启后系统自动将用户已绑定的手机门号带入该栏位。
变更摘要
类型模块说明
A 新增后台 → 系统管理 → 系统参数管理 → 充值提款 → 提款参数配置「是否可以修改及删除银行卡」下方新增「eWallet绑定手机门号」开关，预设关闭
— 不变其余所有后台页面与功能无调整
1.1  充值提款 → 提款参数配置
功能位置：系统管理 → 系统参数管理 → 充值提款 → 提款参数配置，于「是否可以修改及删除银行卡」下方新增「eWallet绑定手机门号」开关。
1.1.1  示意图
后台 — 提款参数配置截图（蓝色高亮行为本次新增，可点击开关切换状态）

1.1.2  新增开关说明
1
开关名称：eWallet绑定手机门号，位于「是否可以修改及删除银行卡」下方一行。
2
样式与既有开关（如「是否可以修改及删除银行卡」）保持一致。
3
预设值为关闭。
1.1.3  开关状态与前台行为对照
开关状态用户手机门号状态前台绑eWallet(含GCash、Maya)「手机号码」栏位行为
开启已绑定手机门号自动带入用户已绑定的手机门号，不可手动修改
开启未绑定手机门号保留可自行手动输入的功能（与开关关闭时行为相同）
关闭任意维持现有行为，用户可自行填写手机号码
1.1.4  储存与生效
1
开关或关闭立即储存其状态。
2
储存成功后前台实时生效。
3
储存成功 / 失败提示与既有提款参数配置操作反馈保持一致。
1.1.5  情境补充
情境 A：本次变动仅影响「手机号码」栏位的带入逻辑，不影响其他绑卡流程。
情境 B：其余「提款参数配置」中的原有开关与规则均不受影响，维持原逻辑。

20260714v1.0A 新增文件初版 — 绑 eWallet 页「手机号码」栏位逻辑调整—
需求目的
当后台开关「eWallet绑定手机门号」启用时，前台绑定 eWallet(含GCash、Maya) 流程中的「手机号码」栏位，自动带入用户已绑定的手机门号且不可更改。
变更摘要
类型模块说明
A 新增前台 → 绑定 eWallet「手机号码」栏位依后台开关状态与用户绑定状态动态处理
— 不变其余绑卡 / eWallet 流程栏位无调整
2.1  eWallet — 手机号码栏位
影响页面：前台绑定 eWallet(含GCash、Maya) 页均含「手机号码」输入栏位。其余栏位不受影响。
2.1.1  逻辑规则
1
后台开关开启 & 用户已绑定手机门号
→ 手机号码栏位自动带入已绑定门号，栏位不可编辑（只读样式）。
2
后台开关开启 & 用户未绑定手机门号
→ 手机号码栏位保持可手动输入状态。
3
后台开关关闭
→ 维持现有行为，用户可自行填写，无自动带入。
4
其余未提及的旧规则无调整。

2.1.3  栏位样式规范
状态栏位样式说明
自动带入（已绑定）只读 / 灰底显示用户手机门号，不可点击编辑；参考既有只读栏位样式
可手动输入一般输入框与现有手机号码栏位样式相同，无任何变化
2.1.4  情境说明
情境 A
后台开关 开启，用户已绑定手机门号

→ 手机号码栏位自动显示已绑定门号，只读，无法修改。
情境 B
后台开关 开启，用户未绑定手机门号

→ 手机号码栏位维持可手动输入。
情境 C
后台开关 关闭，用户状态任意

→ 维持现有行为，可自行填写，无自动带入。
情境 D — 不变规则
其余绑卡 / eWallet 栏位及验证逻辑均不受本次影响，维持原有规则。
