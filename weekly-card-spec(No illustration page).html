頁面總覽
頁面結構、活動時間與週卡效期規則

頁面結構

頁面為 H5 單頁捲動設計，由上至下依序包含四個主要區塊：

  • ① Hero：Banner 圖片 + 活動名稱 + 效期 Pill + 規則入口
  • ② 方案選擇器：Tab 切換標準 / 豪華週卡規格，含方案詳情卡與購買按鈕
  • ③ 簽到進度卡：進度條 + 7 天進度點，可點擊跳轉至對應獎勵卡片
  • ④ 每日獎勵列表：共 7 張卡片，依簽到狀態動態渲染

活動時間與週卡效期

  • 活動時間：後台設定起訖，前台僅顯示文字，不做時間控制邏輯
  • 週卡效期起算：購買完成 API 回傳成功後，當下即計為第 1 天，不等隔日凌晨重置
  • 效期由後台計算：前台從 API 取得效期區間後直接顯示，格式 YYYY/MM/DD — YYYY/MM/DD
  • 週期（7 天）結束後用戶資格自動重置，可購買下一週期
⚠️
特殊情境：用戶於 23:50 購買週卡，當日（購買日）即為第 1 天。即便僅剩 10 分鐘，第 1 天的簽到獎勵仍可在當日 24:00 前領取，過期則不予補發。

3
Hero 區塊
Banner 圖、活動名稱、效期 Pill、規則入口

Hero 組成元素

  • Banner 圖片：後台上傳，以 <img> 渲染，佔滿 Hero 區域（高度固定 190px）。圖片 URL 從 API 取得
  • 活動名稱：疊加於 Banner 上方，固定顯示「週卡活動」
  • 效期 Pill：顯示於標題下方
    • 未購買：顯示活動期間（後台設定）
    • 已購買：顯示個人週卡效期（後台回傳）
  • 規則入口：固定於 Hero 右側，點擊觸發活動規則彈窗（見 §8.5）
ℹ️
Banner 圖片若載入失敗，需有 fallback 處理（建議保留漸層背景色作為預設底色），避免區塊空白。
4
週卡方案選擇器
Tab 切換、每週限購、已購 dimmed 狀態

規格說明

  • 共 2 種規格（名稱、售價由後台配置），設計支援擴充
  • 每個週期用戶只能購買 1 種，購買後同週期內不可切換，也不可重複購買

Tab 切換邏輯

  • 預設選中第 1 種規格
  • 切換 Tab 後，同步更新：方案詳情（文案、獎勵標籤）、簽到進度卡、每日獎勵列表、購買按鈕

已購狀態規則

  • 已購的 Tab 顯示「✓ 已購」綠色標籤
  • 切換至另一規格：整個內容區灰暗（dimmed），購買按鈕顯示「每週限購一次」禁用
  • 切換回已購規格：內容區恢復正常
⚡ 特殊情境 — 用戶誤觸另一規格 Tab

用戶已購標準週卡 → 點擊豪華週卡 Tab → 內容灰暗 + 按鈕禁用 → 點回標準週卡 Tab → 恢復正常。整個過程不呼叫任何 API。

特殊規則

  • 點擊天數點：頁面以 smooth scroll 滾動至對應獎勵卡片，卡片短暫高亮（約 1.5s）後恢復

6
每日獎勵列表
卡片版型、普通獎勵、額外獎勵、過期規則、全狀態對照

卡片版型

  • 版型 A（雙欄）：後台有配置額外獎勵 → 左欄普通獎勵、右欄額外獎勵
  • 版型 B（單行）：後台未配置額外獎勵（null）→ 僅顯示普通獎勵

普通簽到獎勵

  • 支援類型：現金（PHP）、積分（pts）、免費旋轉（次數）、優惠券（張數）
  • 現金（Bonus）類型：需附顯示流水倍數說明（倍數由後台配置）
  • 積分 / 旋轉 / 優惠券：無流水要求，不顯示流水說明

額外獎勵規則

  • 達成條件：僅計算當日（自然日）充值或投注，跨日不累計
  • 條件達成即可領取，無需等待其他事件
  • 過期：當日 24:00 後無論條件達成與否，一律視為過期，顯示「已過期」，不予補發
🚫
普通獎勵同樣有過期限制：須於當日 24:00 前手動領取。過期後按鈕顯示「已過期」禁用，不予補發。整張卡片呈灰暗樣式。

彈窗顯示規則
四種彈窗的觸發時機、內容、關閉方式

8.1 自動簽到提示（Auto Reward Sheet）

進入頁面時自動從底部滑出，提示用戶領取今日獎勵。

  • 觸發條件：用戶已購買週卡 + 今日尚未完成普通簽到
  • 不顯示：未購買週卡 / 今日已簽到

內容：

  • 今日天數標題（例：「第 4 天簽到獎勵」）
  • 今日普通獎勵明細（圖示 + 數值 + 名稱）
  • 額外獎勵區塊（若後台有配置）：
    • 條件未達標 → 顯示獎勵內容 + 條件文字 + 「存款」捷徑按鈕（跳轉存款頁）
    • 條件已達標 → 顯示「領取」按鈕
  • 「確認」按鈕：完成簽到（普通獎勵發放），關閉彈窗，頁面狀態同步

關閉方式：

  • 點「確認」→ 關閉 + 完成簽到
  • 點遮罩 → 僅關閉，不觸發簽到
⚠️
點遮罩關閉後，用戶仍需至每日獎勵列表手動點擊今日的「領取」按鈕才能完成簽到。

8.2 購買確認彈窗

  • 觸發：點擊方案詳情卡「立即購買」按鈕
  • 內容：方案名稱、購買金額、「確認購買」與「取消」按鈕

  • 關閉方式：點「取消」或點遮罩 → 不執行購買
⚡ 特殊情境 — API 購買失敗

購買 API 失敗（餘額不足、網路錯誤）→ 彈窗不關閉，顯示錯誤提示文字，按鈕恢復可點擊狀態。

8.3 每日簽到完成彈窗

  • 觸發：用戶點擊今日獎勵卡片的「領取」按鈕後彈出
  • 內容：天數標題 + 今日普通獎勵明細（由後台配置渲染）
  • 無操作按鈕：簽到已於點擊「領取」時完成，此彈窗僅為確認提示
  • 關閉方式：點右上角「✕」或點遮罩
ℹ️
此彈窗關閉後，頁面狀態已完成更新（今日按鈕已切為「✓ 已領取」），不需再觸發任何 API。

10
活動規則彈窗
固定文字 7 條規則

觸發與關閉

  • 觸發：點擊 Hero 區「活動規則」按鈕
  • 關閉：點「✕」或點遮罩

固定規則文字（共 7 條）

  1. 每個週期僅可購買一次週卡，週期結束後資格自動重置。
  2. 購買成功後當下即計為第 1 天，往後依序至第 7 天；簽到獎勵按實際登入天數累計，若週期結束前未全部簽到，週期結束時強制重置。
  3. 日常簽到獎勵（FREE）登入後可直接手動領取；額外獎勵須滿足當日條件後才可領取。
  4. 彩金（Bonus）類獎勵需完成對應流水倍數之有效投注，積分 / 旋轉 / 優惠券不設流水要求。
  5. 額外獎勵條件僅計算當日充值或投注金額，達成後即可領取；須於當日 24:00 前領取，逾期不可補領，不予補發。
  6. 普通簽到獎勵同樣須於當日 24:00 前手動領取，逾期視為放棄，不予補發。
  7. 活動期間若獎勵規則調整，已領取獎勵不受影響，未領取按最新規則執行。
